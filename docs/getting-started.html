Installation

Maven

Add a test dependency for Cuppa in your project’s POM:

<dependency>
    <groupId>org.forgerock.cuppa</groupId>
    <artifactId>cuppa</artifactId>
    <version>1.0.0</version>
    <scope>test</scope>
</dependency>

To get Surefire (the Maven plugin that runs unit tests) to run Cuppa tests, you’ll need some additional configuration:

<plugins>
    <plugin>
        <groupId>org.apache.maven.plugins</groupId>
        <artifactId>maven-surefire-plugin</artifactId>
        <version>2.19.1</version>
        <dependencies>
            <dependency>
                <groupId>org.forgerock.cuppa</groupId>
                <artifactId>cuppa-surefire</artifactId>
                <version>1.0.0</version>
            </dependency>
        </dependencies>
    </plugin>
</plugins>

If you want to use Cuppa to write integration tests, you’ll need to do the same thing for Failsafe. Add cuppa-surefire as a dependency of the maven-failsafe-plugin plugin.

Gradle

Add a test dependency for Cuppa in your project’s build file:

dependencies {
    testCompile 'org.forgerock.cuppa:cuppa:1.0.0'
}

Plain Old Jar File

Alternatively, you can download binaries for the latest release.

Already got tests?

If your project already contains tests that were written with another testing framework, there are several methods to integrate Cuppa.
